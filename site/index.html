pg_retrieval_engine

FAISS 驱动的 PostgreSQL 向量检索扩展 FAISS-powered vector search extension for PostgreSQL

目标：在满足召回约束下，提供高吞吐 ANN 检索能力（CPU/GPU）。 Goal: deliver high-throughput ANN search on CPU/GPU under strict recall constraints.

GitHub 仓库 GitHub Repository 查看文档 Read Docs

项目目的 Project Purpose

  • 把 FAISS 检索能力集成到 PostgreSQL，提供函数式向量检索接口。
  • Integrate FAISS into PostgreSQL with function-oriented vector search APIs.
  • 以 `vector/vector[]` 作为主通道，降低数据转换开销。
  • Use `vector/vector[]` as the primary path to reduce conversion overhead.
  • 支持 `hnsw` / `ivfflat` / `ivfpq` 及 `l2` / `ip` / `cosine`。
  • Support `hnsw` / `ivfflat` / `ivfpq` with `l2` / `ip` / `cosine` metrics.

快速使用 Quick Usage 

CREATE EXTENSION vector;
CREATE EXTENSION pg_retrieval_engine;

SELECT pg_retrieval_engine_index_create(
  'docs_hnsw',
  768,
  'cosine',
  'hnsw',
  '{"m":32,"ef_construction":200,"ef_search":64}'::jsonb,
  'cpu'
);

SELECT *
FROM pg_retrieval_engine_index_search(
  'docs_hnsw',
  '[0.1,0.2,0.3,...]'::vector,
  10,
  '{"ef_search":128}'::jsonb
);
